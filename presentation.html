What are they?

Key Concepts

Custom Elements
Custom-defined HTML tags that you can use in your documents, whose behavior is described using a standard Javascript class and callback methods
Template Elements (<template>)
Elements that contain HTML that will not be displayed in the DOM by default, but which can be used and/or copied into other elements elsewhere on the page
Slot Elements (<slot>)
Elements that describe where incoming elements should be displayed within a template, particularly in the Shadow DOM (see below)
Shadow DOM
An isolated tree of elements contained within a given custom element. Shadow DOM will not inherit any of the styling of the outer "live" DOM. Shadow DOM can contain <slot> elements describing where "live" DOM children of the custom element should be displayed (more on this later). Shadow DOM is optional!

Countdown Example

Element Description


class BasicCountdown extends HTMLElement {
  constructor() {
    super();
  }

  startCountdown() {
    clearInterval();
    this.displaySeconds();
    this.interval = setInterval(() => {
      if (this.seconds < 1) {
        this.seconds = parseInt(this.getAttribute("seconds"));
        this.displaySeconds();
      } else {
        this.seconds -= 1;
        this.displaySeconds();
      }
    }, 1000);
  }

  clearInterval() {
    if (this.interval) {
      window.clearInterval(this.interval);
    }
  }

  displaySeconds() {
    this.innerHTML = `${this.seconds}`;
  }

  attributeChangedCallback(name, oldVal, newVal) {
    if (name === "seconds") {
      this.seconds = parseInt(newVal);
      this.startCountdown();
    }
  }

  static get observedAttributes() {
    return ["seconds"];
  }
}

window.customElements.define("tts-countdown-basic", BasicCountdown);

Custom Elements

Let's break it down!

  • Custom elements are defined using the class syntax, subclassing the built-in HTMLElement class:

    class MyCustomElement extends HTMLElement {
  • HTMLElement has specific lifecycle methods that you can override in order to establish specific behaviors (more on this in a second)

  • You tell the DOM about the existence of your custom element by mapping an element name to the class you've defined, using window.customElements.define('my-element', MyCustomElement).

    This will allow you to use <my-element> on your page

  • Element names must have a token with a dash after it -- this distinguishes them from the built in html5 elements.
